# Network Security Group (NSG)

VNet 내 Azure 리소스 간 네트워크 트래픽을 필터링. 인바운드/아웃바운드 트래픽을 허용하거나 거부하는 보안 규칙의 모음으로, 서브넷 또는 NIC에 연결하여 적용

## 보안 규칙

각 NSG 규칙은 아래 속성으로 구성되며, 우선순위 순서대로 평가되고 규칙과 일치하면 이후 규칙은 처리하지 않음

| 속성 | 설명 |
|------|------|
| 이름 | NSG 내에서 고유한 이름 |
| 우선순위 | 100~4096 사이의 숫자. 낮을수록 먼저 처리되며, 일치하는 규칙이 있으면 이후 규칙은 평가하지 않음. 기본 규칙은 가장 높은 숫자(낮은 우선순위)로 지정되어 사용자 지정 규칙이 항상 먼저 처리됨 |
| 원본/대상 | Any, 개별 IP, CIDR(예: 10.0.0.0/24), 서비스 태그, 애플리케이션 보안 그룹 지정 가능. 단일 규칙에 여러 IP/범위 지정 가능하나 여러 서비스 태그나 애플리케이션 보안 그룹은 불가 |
| 프로토콜 | TCP, UDP, ICMP, ESP, AH, Any |
| 방향 | 인바운드 또는 아웃바운드 |
| 포트 범위 | 단일 포트(80), 범위(10000-10005), 혼합(80, 10000-10005) 지정 가능 |
| 작업 | Allow 또는 Deny |

### 기본 인바운드 규칙

NSG 생성 시 자동으로 포함되는 기본 규칙. 삭제 불가하며 사용자 지정 규칙보다 낮은 우선순위(높은 숫자)로 항상 마지막에 평가됨

| 이름 | 우선순위 | 원본 | 원본 포트 | 대상 | 대상 포트 | 프로토콜 | 작업 |
|------|--------|------|---------|------|---------|--------|------|
| AllowVNetInBound | 65000 | VirtualNetwork | 0-65535 | VirtualNetwork | 0-65535 | Any | Allow |
| AllowAzureLoadBalancerInBound | 65001 | AzureLoadBalancer | 0-65535 | 0.0.0.0/0 | 0-65535 | Any | Allow |
| DenyAllInbound | 65500 | 0.0.0.0/0 | 0-65535 | 0.0.0.0/0 | 0-65535 | Any | Deny |

### 기본 아웃바운드 규칙

인바운드와 달리 `AllowInternetOutBound` 규칙으로 인해 기본적으로 인터넷 아웃바운드가 허용됨

| 이름 | 우선순위 | 원본 | 원본 포트 | 대상 | 대상 포트 | 프로토콜 | 작업 |
|------|--------|------|---------|------|---------|--------|------|
| AllowVnetOutBound | 65000 | VirtualNetwork | 0-65535 | VirtualNetwork | 0-65535 | Any | Allow |
| AllowInternetOutBound | 65001 | 0.0.0.0/0 | 0-65535 | Internet | 0-65535 | Any | Allow |
| DenyAllOutBound | 65500 | 0.0.0.0/0 | 0-65535 | 0.0.0.0/0 | 0-65535 | Any | Deny |
